# EduProgram - Анализатор образовательных программ

## 1. Описание проекта

**EduProgram** - веб-сервис для автоматизированного сравнения и анализа образовательных программ на основе рабочих планов университетов.

Сервис позволяет:
- загружать данные о программах из различных источников (Wikidata, сайты университетов, ручной ввод);
- визуализировать структуру программы в виде графа «Курс — Пререквизиты — Постреквизиты — Связанные темы»;
- сравнивать свою программу с программами ведущих университетов;
- выявлять пробелы, избыточность и критически важные дисциплины («мосты»);
- получать рекомендации по улучшению программы;
- экспортировать результаты в различные форматы.

Проект реализуется в формате веб-приложения;

---

## 2. Основная проблема
Модернизация образовательных программ вручную — сложный и трудоемкий процесс, требующий анализа большого объема данных из разных источников. Методисты и преподаватели сталкиваются со следующими трудностями:
- отсутствие инструментов для наглядного сравнения рабочих программ;
- сложность выявления пробелов и избыточности в содержании.

EduProgram автоматизирует процесс анализа, предоставляет наглядную визуализацию структуры программ в виде графа и на их основе формирует рекомендации по улучшению.

---

## 3. Категории пользователей
## Преподаватель/методист (основной пользователь)
Характеристика:
- отвечает за разработку и модернизацию образовательных программ;
- имеет базовый или продвинутый уровень работы с ПК, но не является программистом или аналитиком данных;
- работает с учебными планами, нормативными документами;
- заинтересован в выявлении слабых мест и избыточностей программы;
- нуждается в наглядном представлении структуры программы и ее сравнении с ведущими университетами.

Важно: 

- понятная визуализация графа курсов, тем и учебных единиц с возможностью фильтрации и поиска;
- получение рекомендаций: что добавить, что убрать, какие курсы являются «мостами»;
- экспорт результатов в наглядные форматы: `pdf` или `docx` для отчета и `rdf` для графа.

---

## 4. Постановка задачи
Разработать веб-сервис, который:

**Позволяет загружать данные:**

- из Wikidata (навыки, знания, темы по ключевому слову);
- о рабочих программах университетов (автоматически по списку топ-20 или вручную);
- о своей образовательной программе (ручной ввод).

**Выполняет:**

- построение графа «Курс — Пререквизиты — Постреквизиты — Связанные темы»;
- наложение графов разных университетов для сравнительного анализа;
- выявление «мостов» — дисциплин, присутствующих во всех учебных планах;
- кластеризацию для выделения специализаций;
- поиск учебных единиц по графу.

**Генерирует:**
- граф с количественными метриками вхождения учебных единиц;
- текстовые рекомендации по улучшению программы (что добавить, что убрать).

**Позволяет:**

- переключать режимы отображения (полный граф, мосты, кластеры, рекомендации);
- экспортировать граф в формате `rdf`;
- экспортировать отчеты и рекомендации в `pdf` и `docx`;
- задавать университеты для сравнения вручную.

## 5. Функциональные требования:
**5.1 Визуализация графа учебной программы**

Система должна отображать граф, вершинами которого являются:
- Направление подготовки;
- Дисциплины;
- Темы в рамках дисциплин;
- Учебные единицы;
- Пререквизиты;
- Постреквизиты.

**5.2 Сравнительный анализ и метрики**
- Наложение графов: визуальное сравнение структуры программ разных университетов;
- Количественная мера: отображение количества вхождения определенной учебной единицы в программы других вузов (цветовое выделение или численное значение).

**5.3 Режимы отображения и аналитика**

Пользователь должен иметь возможность переключаться между режимами:
- Полный граф с количественными показателями вхождения;
- Отображение «мостов»: выделение дисциплин, присутствующих во всех загруженных учебных планах;
- Кластеризация: выделение кластеров-специализаций (с применением алгоритмов кластеризации графов).

**5.4 Формирование рекомендаций**
- Автоматическое формирование текстовых рекомендаций по улучшению программы на основе анализа графа (что стоит добавить, а что убрать).

**5.5 Экспорт результатов**

- Экспорт графа: выгрузка в формате `rdf` для последующего импорта в другие программы;
- Экспорт отчетов: выгрузка текстовых рекомендаций в форматы `pdf`, `docx` и др.

## 8. Сценарии использования
## 8.1 Просмотр программы определенного вуза

Действующее лицо: Неавторизованный пользователь

Предварительные условия: Пользователь находится на главной странице

Основной сценарий:

1. Пользователь нажимает на кнопку загрузки файлов
2. Пользователь загружает файлы направлений интересующего вуза
3. Приложение показывает дерево, узлами которого являются предметы показанные в порядке изучения

## 8.2 Просмотр программы нескольких вузов

Действующее лицо: Неавторизованный пользователь

Предварительные условия: Пользователь находится на главной странице

Основной сценарий:

1. Пользователь нажимает на кнопку загрузки файлов
2. Пользователь загружает файлы направлений интересующих вузов
3. Приложение показывает дерево, узлами которого являются предметы показанные в порядке изучения. Совпадающие, в разных вузах, предметы накладываются друг на друга, несовпадающие выделяются

## 8.3 Просмотр мостов 

Действующее лицо: Неавторизованный пользователь

Предварительные условия: Загружены рабочие программы, пользователь находится на главной странице

Основной сценарий:

1. Пользователь переходит во вкладку мосты
2. Приложение показывает мосты дерева, выделяя их цветом

## 8.4 Просмотр рекомендаций

Действующее лицо: Неавторизованный пользователь

Предварительные условия: Загружены рабочие программы, пользователь находится на главной странице

Основной сценарий:

1. Пользователь переходит во вкладку рекомендации
2. Приложение показывает рекомендации, составленные на основе анализа графов рабочих программ (что стоит убрать, что стоит добавить)


## 8.5 Экспорт результатов анализа

Действующее лицо: Неавторизованный пользователь

Предварительные условия: Загружены рабочие программы, пользователь находится на главной странице

Основной сценарий:

1. Пользователь нажимает на кнопку экспорта графа
2. Всплывает диалоговое окно проводника, где пользователь выбирает путь сохранения файла
3. Граф сохраняется по указанному пути
4. Пользователь выбирает формат экспорта отчета, нажав на соответствующую кнопку
5. Всплывает диалоговое окно проводника, где пользователь выбирает путь сохранения файла
6. Отчет сохраняется по указанному пути
